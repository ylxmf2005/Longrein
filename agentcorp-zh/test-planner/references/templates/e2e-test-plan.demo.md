# E2E 测试手册：示例标题

## 执行形式

- 默认：浏览器操作作为主要证据（真实浏览器，已登录会话）；截图和页面状态是主要证据，API/DB/日志是辅助。
- 降级：仅当在此明确声明时；说明降级后不能再证明什么。

## 前置条件

- 入口 URL、登录方法（凭证仅引用）、测试数据准备（精确到对象 ID；在上下文中未找到的在此填入）。

## FLOW-1 (P1)：用户目标名称

- 角色：高级用户。覆盖：FR-x / AC-y。

| # | 页面/位置 | 动作（输入逐字给出） | 预期行为 | 证据 |
|---|---|---|---|---|
| 1 | 任务列表页 `<URL>` | 点击"新建任务" | 创建对话框弹出 | 截图 |
| 2 | 创建对话框 | 在 prompt 输入框中输入：`Help me translate the README into English` | 提交后跳转到任务详情页 | 截图，记录 task_id |
| 3 | 任务详情页 | 等待执行完成 | 状态变为成功，产物可下载 | 截图；辅助证据 `GET /api/task/<id>` |

- 错误路径（也要逐步编写）：

| # | 页面/位置 | 动作 | 预期行为 | 证据 |

| 1 | 创建对话框 | 留空 prompt 提交 | 行内错误，未发送请求 | 截图，网络面板 |
