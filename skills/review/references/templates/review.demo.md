# Review 产物 Demo

`review.md` 保存对准确对象和基线的当前独立裁决。表达随风险压缩；裁决、findings、审查边界和证据限制保持可见。

```markdown
# Review：<对象>

## 裁决

- 结论：approve | request_changes | needs_more_evidence
- 对象：<准确版本、分支、diff、提交或产物>
- Baseline：<实际比较基线或不适用原因>
- 核心理由：<什么证据决定当前对象能否前进>

## Findings

### [P0-P3] <标题>

- 位置：<文件与行、章节、接口或状态>
- 触发与路径：<具体输入、调用者、扰动和实际经过>
- 后果：<错误结果、破坏的承诺或长期成本>
- 证据：<代码、命令、请求、日志、规则或运行结果>
- 与当前对象的关系：<本次引入、放大，或会改变当前裁决的既有事实>
- 处理方向：<正确修复落在哪里，以及什么仍需决定>
- 证据边界：<尚未确认的部分>

## 审查边界与剩余风险

- 已检查：<真实检查过的变化面、调用者与产物>
- 未检查：<未覆盖对象及原因>
- 剩余风险：<裁决后仍需承担的风险或证据缺口>
```

没有 findings 时删除示例 finding，用一段话说明审查范围、证据边界和未发现阻断问题的依据。
