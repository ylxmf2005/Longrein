# 证据驱动的长期变化

长期规则会影响许多尚未发生的任务。行为变化从真实轨迹开始：什么输入触发了什么判断和行动，产生了什么后果，用户怎样纠正；同时保留仍然有效的行为和能够暴露过拟合的相邻反例。

先允许不同原因解释同一现象。形成可以验证的原因后，再固定代表性输入、模型、工具和环境，只改变准备验证的规则或语义锚点，重放原轨迹与相邻反例。比较正确性、完整性、用户介入、意外动作、成本与长上下文副作用。

证据支持哪一层原因，变化就落在哪一层。触发发现、判断、权限、动作或产物语义发生变化时使用行为重放；引用、格式、链接和 UI 元数据等机械问题使用静态检查。

用户批准针对具体变化范围或 diff。批准研究问题、认可一次纠正或调用 Evolution，不自动授权修改长期行为。

落地后重新运行相同证据。文字改变可以称为已修改；前后结果与反例共同支持未来行为更好时，才能称为已改善。
