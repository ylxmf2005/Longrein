# grill

## 把方向展开成决策树

从 `context.md` 和 Current Artifacts 进入真实对象，持续访谈用户，直到双方对当前承诺形成共同理解。把方向看成一棵决策树：每个决定下面挂着依赖它的决定。

默认用户没有读过当前产物。一个问题需要背景才能判断时，先讲清相关关系、推荐答案及其代价，再请用户决定；不把阅读文档或补查项目事实转交给用户。

## 一轮推进一层前沿

前沿是前置决定已经成立、现在不需要猜测就能回答的全部问题。每轮列出整个前沿，为问题编号并给出推荐答案，然后等待用户回答。依赖本轮其他未决答案的问题留到下一轮。

用户的回答会重塑决策树。每轮之后重新计算前沿，让已经成立的决定放出下一层问题，不沿旧问卷机械继续。

能从代码、运行现场、文档或工具中查明的事实由 Agent 调查。调查中的事实只挂起依赖它的分支，其他前沿继续；需要用户提供的是价值、偏好、代价接受、范围、风险与授权。

## 取得共同理解再继续

当与当前承诺有关的承重分支都已触及，没有关键决定被默默假定，并且用户确认双方已经形成共同理解，Grill 才结束。在此之前不开始实施，也不把用户的困惑或“我不知道”当作同意。

回答推翻了 Goal、Scope、Non-goals 或专业产物时，回到 `shape` 修订权威来源；方向仍成立且用户决定继续时交给 `dev`。Grill 默认不创建产物，也不替用户批准方向、替 `review` 独立证伪或替 `test` 提供完成证据。
