# Diagnosis 产物 Demo

这份文件是 `shape/diagnosis.md` 的唯一结构来源。只有根因需要独立交接时创建；不复制 `task.md` 中的任务承诺。

```markdown
# Diagnosis：<故障或问题>

## 被诊断对象

- 触发与可观察故障：<具体输入、前置状态和结果>
- 对象与版本：<实际检查的系统、代码、配置或数据>
- Task Context：context r<revision> / scope r<revision> | not-applicable

## 因果链

<具体触发> → <状态、数据或控制流传播> → <错误机制> → <最终可观察结果>

## 根因

<哪个机制足以解释故障，以及为什么它是根因而不是最后一层症状。>

## 促成条件与影响范围

- 促成条件：<会放大或暴露问题、但不是根因的条件；没有则写 none>
- 受影响入口与调用者：<实际可达表面>
- 数据、状态与下游结果：<传播范围>
- 明确不受影响：<检查依据；没有依据时不要填写>

## 反证与替代解释

- 已排除：<候选解释及推翻证据；没有则写 none>
- 尚未排除：<缺少什么证据、会改变哪项结论；没有则写 none>

## 证据

- <代码、日志、请求、数据、复现或运行结果及其支持边界>

## 修复将触及的结构

<只说明正确修复需要改变的系统层次、边界或契约；用户未授权实现时不写成已决定方案。>

## 结论边界与下一步

- 已确定：<当前能够诚实成立的诊断结论>
- 仍缺：<证据或 Owner 决定；没有则写 none>
- 下一位：<owner、shape、dev、review、test 或其他>
```
