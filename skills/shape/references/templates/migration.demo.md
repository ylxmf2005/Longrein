# Migration 产物 Demo

这份文件是 `shape/migration.md` 的唯一结构来源。只有数据、状态、接口或部署形态需要独立的过渡、混合版本和退出判断时创建；它定义迁移模型，不复制 `plan.md` 的工作进度。

```markdown
# Migration：<对象或变化>

## 来源与迁移边界

- Task Context：context r<revision> / scope r<revision> | not-applicable
- Requirements：<相关 REQ-*>
- Design decisions：<相关 AD-*>
- 迁移对象：<数据、状态、接口、配置或部署单元>
- 规模与现实约束：<已确认的数据量、并发、窗口和平台行为>

## 起点、终点与不变量

- 起点：<迁移前真实状态>
- 终点：<迁移后必须成立的状态>
- 全程不变量：<任何阶段都不能破坏的行为、数据和权限>

## 过渡模型

- 准备：<schema、兼容读取、影子路径或前置能力>
- 转换：<回填、双写、切流、重算或状态转换>
- 混合版本：<不同版本、批次或消费者共存时的语义>
- 收敛：<新形态成为唯一事实源的条件>
- 退出：<兼容层、旧字段、旧入口或开关的删除条件>

## 失败与恢复

- 部分成功：<怎样识别、重试、补偿或人工恢复>
- 回滚：<允许回滚的边界、数据影响和停止条件>
- 不可逆点：<何时产生以及需要什么确认>

## 可观察性与验证入口

- 进度与终态：<日志、指标、数据回读或审计>
- 聚焦检查：<每个关键过渡应证明什么>
- 外部证明：<Test 或运行验证入口>

## 开放问题与 Plan 影响

- <会改变迁移策略、Phase 依赖或发布条件的问题；没有则写 none>
```
