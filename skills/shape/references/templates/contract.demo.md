# Contract 产物 Demo

这份文件是 `shape/contract.md` 的唯一结构来源。只有公共或跨模块契约需要被多个消费者精确引用、独立评审和持续修订时创建；`design.md` 链接它，不再维护第二份契约正文。

```markdown
# Contract：<接口、事件或数据边界>

## 来源与适用范围

- Task Context：context r<revision> / scope r<revision> | not-applicable
- Requirements：<相关 REQ-*>
- Design decisions：<相关 AD-*>
- Owner：<契约拥有者>
- Consumers：<已确认的调用者、下游或用户入口>

## 契约定义

- 输入：<字段、类型、必填、默认值和合法范围>
- 输出：<成功结果、字段和可观察语义>
- 状态与事件：<合法状态、转换、顺序和幂等语义>
- 错误：<拒绝、冲突、重试、依赖失败和不可恢复失败>
- 权限：<身份、授权、资源所有权和敏感数据边界>

## 兼容与演进

- 必须保持：<现有消费者依赖的行为>
- 允许改变：<内部实现或明确授权变化的表面>
- 混合版本：<新旧调用者怎样共存>
- 退出条件：<旧契约何时、由谁、依据什么停止支持>

## 示例与反例

<只保留能够消除真实歧义的请求、响应、事件或状态示例；没有则写 none。>

## 开放问题与验证入口

- 开放问题：<会改变契约的问题和 Owner；没有则写 none>
- Dev 聚焦检查：<实现期间的契约检查>
- Review / Test 入口：<独立审查和外部证明表面>
```
