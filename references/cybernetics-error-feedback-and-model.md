# 从控制论看 Skill：误差、必要多样性与内部模型

> 原始来源：W. Ross Ashby, *An Introduction to Cybernetics*（1956）中关于 [Requisite Variety 的原文摘录](https://www.panarchy.org/ashby/variety.1956.html)，以及 Roger C. Conant 与 W. Ross Ashby, [“Every Good Regulator of a System Must Be a Model of That System”](https://pespmc1.vub.ac.be/books/Conant_Ashby.pdf)（1970）。本文把控制论用于 Skill 设计的类比；类比用于提出可验证实践，不把它冒充 LLM 内部机理证明。

## 一、控制不是发命令，而是让关键变量留在可接受区域

Ashby 把调节抽象为四个部分：

- `D`：外界扰动；
- `R`：调节器可采取的反应；
- `T`：扰动与反应共同作用的环境；
- `E`：必须保持在可接受范围内的关键变量。

调节器的目标不是让世界停止变化，而是在多种扰动下仍让 `E` 落入可接受集合。

映射到 Agent 工作，用户提示词不等于完整控制命令。真正的目标是可观察结果：功能正确、权限不越界、重要行为不回归、用户能够理解并作出下一次判断。代码库、工具、依赖和新证据会持续制造扰动；Skill 的作用是帮助模型根据这些差异调整行动。

## 二、必要多样性：不能用一种固定反应处理所有未知

Ashby 的必要多样性定律说明，调节器能够达到的调节能力，不会超过它能够接收和区分的信息，以及能够产生的反应多样性。简化说，只有足够的多样性才能吸收多样性。

这并不意味着 Skill 要写更多规则。恰恰相反，如果一个 Skill 无法区分任务状态，再多同义禁令也只是同一种反应。

Shape 中的调查、原型、提问、压力测试，价值不在于形成四步流程，而在于它们对应四种不同信息状态：事实未知、偏好隐性、决定未作出、方案前提脆弱。调节器先识别状态，再选择反应，才拥有真正的多样性。

## 三、Good Regulator：有效调节依赖一个足够真实的内部模型

Conant 与 Ashby 在明确假设下证明：最简单的最优调节器，其行动必须与被调节系统存在映射关系；换句话说，它必须以某种形式成为系统的模型。论文也强调，环境统计随时间变化时，最佳调节器需要随之改变模型。

对 Skill 的启示不是“模型必须复制整个仓库”，而是：它必须掌握足以决定当前行动的结构。若不知道真实调用链、权限边界、用户价值和运行反馈，就无法只靠更强语气稳定调节任务。

这支持“地图不是领地”：用户输入是初始模型，代码与现场是被调节对象。Shape 的第一责任是修正内部模型，而不是把初始地图整理得更漂亮。

## 四、具体误差为何比“自行改进”更有用

没有可观察误差时，让模型“把 Skill 变得更好”缺少方向信号。它只能根据语言习惯生成更完整、更严谨或更漂亮的文本，却无法知道哪种行为需要改变。

Bench、失败测试、真实 trace 或用户明确指出的错误，为闭环提供了误差信号：预期结果与实际结果之间的差异。模型可以沿着具体输入、决策和输出路径定位：哪条规则缺失，哪条规则过强，哪个语义锚点引发外溢，哪个工具边界不清楚。

因此，Skill 进化应从同一组失败样本出发，一次修改一组承重指令，再重放相同样本。没有误差的改写只能算设计假设。

## 五、但成熟系统不能永远等错误发生

Conant 与 Ashby 的论文区分了 error-controlled 和 cause-controlled regulation。误差控制在偏差已经出现后才反应，因此通常会留下残余波动；更高级的调节直接利用扰动原因的信息，在关键变量真正偏离前采取行动。

这对“误差优先”形成一个重要修正：

- 第一次改进 Skill 时，真实错误是最可靠的学习入口；
- 找到原因后，应把经验升级为对原因或边界的提前感知；
- 同一种错误不应成为每次运行都必须重现的仪式。

例如，一次性指令被错误写进长期 `AGENTS.md` 后，不能只增加“以后记得删除”的补救步骤。更好的长期控制是分开临时提示与永久规范的写入权限，并要求持久化规则拥有明确的证据和人工批准。

## 六、探索与执行是不同控制档位

探索阶段的目标是扩大可见状态空间：寻找未知、制造原型、比较参考、允许不同假设竞争。过早把路径写死，会减少调节器能够采取的反应。

执行阶段的目标是保持关键变量：确定的范围、权限、安全不变量、兼容性和验收标准需要稳定。此时显式规范并不是敌人，它们定义了不可越过的区域。

当新证据证明内部模型过时，系统应重新开放搜索；方向稳定后再收敛。这比“永远自由”或“永远按计划”都更符合反馈控制。

## 七、Rewind 是控制器状态重置，不是简单撤销

长上下文会保存旧假设、旧待办和已经失效的行动倾向。即使新的事实已经出现，控制器仍可能沿旧轨道输出反应。

Rewind 的价值是把状态拆开：保留可靠观测、失败样本、有效代码和外部事实；清除基于错误模型形成的计划与语义惯性；回到错误前提成为控制策略之前的节点。回溯后的第一步应重新建立反馈信号，而不是立刻继续执行。

## 八、这套类比的边界

控制论定理有自己的数学假设，LLM、用户和软件项目并不自动满足这些条件。“Skill 是调节器”是一种设计模型，不是关于神经网络内部运行的实验证明。

它的价值取决于能否导出可测实践：是否减少同类失败；是否在新证据下更快转向；是否降低无用规则；是否让临时约束停止污染长期记忆。类比不能替代 benchmark。
