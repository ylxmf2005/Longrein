# Plan 产物 Demo

任务根目录 `plan.md` 保存当前有效路线，不复制需求、设计和专业报告。

Plan 只规划真实对象和已有 Diagnosis、Requirements、Design、Contract 或 Migration 能够支撑的工作；执行者仍需自造范围、系统边界或完成标准时，继续 Shape。

路线按真实依赖展开，每一段都通向独立可检查的结果。尽早接触高风险未知和高传播成本，使反馈仍然便宜，过程可理解、可恢复、可验证。

Plan 保存当前有效模型，不保存失效历史。实现反馈先修订拥有结论的源产物，再同步受影响路线；改变用户承诺必须先取得用户决定。

```markdown
# Plan：<任务或变化>

## 来源与当前策略
- Requirements：<路径或不适用依据>
- Design / Diagnosis：<路径>
- 状态：active | blocked | completed | superseded
- 推进策略：<怎样尽早取得反馈并控制传播成本>

## 路线：<独立结果>
- 状态：pending | in_progress | blocked | completed | superseded
- 完成后成立：<事实、能力、产物或验证结论>
- 来源：<REQ、所属专业产物、根因或验证标准>
- Owner：<承担者>
- 依赖：<前置结果或条件>
- 必须保持：<契约和禁止区域>
- 完成检查：<证据入口与通过边界>
- 反馈作用：<什么结果会修订后续路线>
- 当前结果：<简短结论和证据链接>

## 传播与开放问题
<只记录会影响依赖、契约、迁移、权限、用户行为或下一次判断的问题。>

## 当前交接
<接下来要取得的结果、应先读取的来源和最近的反馈入口。>
```
