# 拷问透镜

仅使用适合当前模糊度的透镜。不要把整个列表扔给赞助方。

- **问题真实性**——什么可观察的痛苦、变通办法、成本或需求能证明这件事值得关注？
- **具体受益者**——谁最先受益，以及什么工作变得更容易？
- **反事实**——如果什么都不变会怎样？
- **最小有价值版本**——解决真实问题的最小东西是什么？
- **依附检查**——我们是否依附于某种解决方案的形状而非结果本身？
- **范围边界**——什么诱惑性的相邻工作超出范围？
- **失败模式**——即使实现得很好，为什么仍可能失败？
- **权衡诚实**——当这件事变好时，什么会变糟？
- **耐久性**——产品、政策、基础设施或使用的什么可能变化会让这件事变得更弱？

一次只问一个探针。如果赞助方无法回答，将不确定性记录为假设或开放问题，而非默默解决它。
