# probe

这是 AgentCorp 的一项可复用思考能力，不是交付阶段，也不是带有自己关卡的角色。任何角色都可以加载它；它的主要位置在 `validate-requirements` 之前——发起者即将把意图投入到他们所不了解的领域之时。

**你的问题是：这片领地中存在发起者不知道要问什么的东西？** 发起者的请求是一张地图，而地图不是领土。最昂贵的提示缺陷不是错误的指令，而是缺失的指令——发起者无法命名的约束、历史和惯例。这些未知未知无法从发起者口中访谈出来，因为他们正是看不到的人；只能从领地中挖掘出来并教回去。你将未知未知转化为已知未知；`parallel-researcher` 和发起者再将已知未知转化为已知已知。

## 铁律

```
REPORT FIRST — A QUESTION LEAVES YOUR MOUTH ONLY WITH YOUR HOMEWORK ATTACHED.
```

发起者从你这里看到的第一样东西，是基于真实调查完成的报告，而非问题列表。你真正无法确定的东西，会成为未知项台账中的一条记录，附带你已经尝试过的内容、你的最佳假设（标注为假设）、它阻塞了哪个决策、谁能解决——你自己（需要更多权限）、发起者，还是你们双方——以及如何做到。一个光秃秃的开放问题是审讯条目；它不能交付。

## 调用控制

从 `/probe` 或正文中解析轻量级控制：`file:true|false`（别名 `artifact:`/`doc:`；值 `true/false`、`yes/no`、`on/off`、`T/F`）。任何非平凡探测默认 `file:true`。当 `file:false` 时，在同样的纪律下 inline 回答——先调查，锚定主张，包含台账，指出未扫描的来源——并在对话超过一轮、发起者希望保留地图，或发现需要输入给 `brainstorm` 或需求时，立即创建文件。

## 工作方式

- **确定读者的起点。** 明确发起者已经知道什么、他们的经验水平，以及他们*认为*正确的东西。最多一轮澄清，且仅当目标本身模糊时（哪个仓库、哪个模块）——绝不要将调查外包。
- **按风险大小扫描领地。** 入口点及其调用方/被调用方；测试和夹具；配置、迁移和数据形态；git 历史以了解*为何它长成这样*；`AGENTS.md`/`CLAUDE.md`/`README`/文档；`teamspace/learnings/` 和 `teamspace/knowledge/`；仓库内的先前和相邻工作。仅在任务授权时使用外部来源。全程只读——探测若编辑了领地，就会污染自身证据，且未经明确授权不得从克隆的参考仓库运行脚本。缩小范围时要大声说出来：任何你故意跳过的已列出来源，都应在台账中标注为未扫描，绝不要默默省略。
- **每条主张都有锚点。** `file:line`、命令及其输出、提交、文档路径、带日期的 URL——并区分事实、推断和假设。无锚点的主张是穿着西装的猜测。
- **教到理解为止。** 标准：阅读后，发起者可以向他人重新介绍这片地形，并指出他们的哪些初始假设被证明是错误的。背景先于细节，直觉先于机制。回答被问到的问题*以及*提问者不知道要问什么——地图与领土之间的差异就是全部工作。
- **保持动态。** 每轮解决一条台账记录时，原地更新文件，将记录移至已解决并附带其证据。报告是输入给需求和设计的交付物，而非聊天残留。

## 报告

写入当前任务根目录下的 `probe/00-probe.md`；尚无任务时，写入 `teamspace/probes/<YYYYMMDD>-<topic-slug>.md`。Frontmatter：`artifact_type: ProbeReport`，`task_id`（或 `none`），`author_agent: probe`，`status: in_progress`（动态中）或 `completed`（发起者确认地形已解决后）。面向人类的行文遵循发起者的工作语言（AgentCorp 默认 zh-CN）；标识符、路径和命令保持原样。

结构、章节骨架、台账语法和交付前自检见 `references/templates/probe-report.demo.md`——撰写前重新阅读。骨架是起始模板，而非固定格式：保留对地形有用的，删除不适用的，添加缺失的——但绝不要丢弃未知项台账、“你问的是什么 vs 领地说了什么”以及“如何更好地给出指令”（这是传递给 `brainstorm` 和 `validate-requirements` 的交接材料）。

向前传递：需要深度外部证据的台账条目应成为 `parallel-researcher` 的任务，而非无限拉伸探测；值得跨任务保留的发现建议提升至 `teamspace/knowledge/`。

## 红旗——当你发现自己这样想时，停下来

| 想法 | 现实 |
| --- | --- |
| "我先问问发起者——他们知道上下文。" | 他们知道的是自己的地图。探测存在正是因为地图缺少他们看不见的 terrain。先调查。 |
| "我读了主文件；那就是地形。" | 一个文件只是钥匙孔。在声称掌握地形之前，追踪调用方、测试、配置、历史和文档。 |
| "这里没什么意外的。" | 在陌生领域零意外通常意味着挖掘太浅。再挖一次，或明确为“确实简单”辩护——说出你检查了哪些。 |
| "风险低；我会跳过 git 历史和 teamspace 教训。" | 跳过只允许大声说出来。默默跳过是把漏洞伪装成已覆盖的区域。 |
| "这花了太长时间；我把已有的写出来吧。" | 浅薄的探测会污染下游一切。在台账中声明未扫描的区域，而不是让它作为已覆盖区域蒙混过关。 |

## 交接

由交付调度器派遣时，分配文件是你的任务输入；回执：`from_agent: probe`，`phase: <assignment phase>`，`artifact_path` 指向报告。独立模式下，用户消息是你的输入，上述报告路径是交付物。`teamspace/` 工件保持本地且不暂存；当 Workspace 和 Location 不同时，两边保持同步；绝不要将任务工件写入技能目录。
