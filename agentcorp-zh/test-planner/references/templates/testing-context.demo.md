# 示例项目测试上下文

## 入口和访问

- 预发环境：`https://example.local/app`；登录方法：已登录的 Chrome 会话（凭证仅引用，无秘密）。

## 页面地图

- 任务列表页 `/app/tasks` —— 产品的主入口；核心操作：创建任务、搜索、打开详情。
  - 任务列表页 --点击"New"--> 创建对话框
  - 任务列表页 --点击任务行--> 任务详情页 `/app/tasks/<id>`
- 任务详情页 —— 查看执行状态和产物；核心操作：下载产物、重新运行。

## 核心用户流程

- FLOW：从零创建任务并获得产物（`实际已走过`，2026-06-03）
  - 前置条件：已登录；不需要已有数据。
  - 步骤：在任务列表页点击"New" → 在对话框中输入 prompt → 提交并跳转到详情页 → 等待状态变为成功 → 下载产物。
  - 陷阱：当 prompt 为空时提交按钮变灰，所以自动化必须先断言它是可点击的。

## 可观察表面

- API：`/api/tasks` 系列；响应是统一的 BaseResponse（trace_id/code/message/data）。
- DB：MDB 只读；按 task_id 查询 `tasks` 表。
- 日志：按 trace_id 搜索 CLS。
