# shape

## 先让领地说话

用户的请求、计划和当前上下文是一张地图；代码、运行现场、真实用户和现实约束才是领地。不要先把地图整理成答案。接触真实对象，让事实、矛盾、可能性、代价与仍未看清的部分共同显形，再判断问题是什么。

调查到哪里，结论才能代表到哪里。能从领地取得的事实自行取得；只有价值、偏好、代价接受和授权属于用户。给出明确的专业推荐，在证据反对原方向时直接说明，但不替用户作价值决定。

## 让方向取得承诺资格

探索保持开放，直到继续调查已不再实质改变选择，关键代价已经看清，可观察结果、不能破坏的行为和证据要求也能够说明。

此时收敛方向，把实现路径留给后续现实反馈。调查可以超出当前执行范围；发现更大机会或问题不等于获得实施授权。

方向不是永久权威。新证据推翻承重前提时，重新开放受影响的决定；当前上下文本身已经被旧前提污染时，使用 `rewind` 恢复可信状态。

## 形成当前任务的承诺

当前 Session 已宣布 Active Task 时，读取其精确 `task.md` 路径。收敛时更新其中的 Goal、Scope、Non-goals、Completion Evidence、Must Preserve 和 Task Operating Envelope；这些内容必须来自真实项目和用户决定，不从理论最坏情况生成。

任务涉及代码或分支时，澄清 `source_ref`、`working_branch` 和 `target_ref` 的意图；现场状态与意图不一致且会改变结果时，把事实和代价交给用户确认，不从 checkout 反推承诺。

没有 Active Task 时在当前对话中维护 Shape 结果；用户显式启动 Task 后，Shape 只写入 `task` 已经宣布的任务目录。Shape 不自行创建或选择 Task，也不维护平行状态文件。

用户可以改变 scope；Agent 也可以主动提出更诚实的扩大、收缩或改道。后者在用户同意前只是候选。事实变化增加 `context_revision`；承诺变化同时增加 `scope_revision`，并在 `task.md` 标出哪些旧产物需要重新核验。

## 长程探索

探索跨越多个决定或会话、当前上下文不足以保存可靠状态时，读取 [open-territory.md](references/open-territory.md)。

## 产物

当前 Session 有 Active Task，且方向比较、代价或关键未知需要交接时，才保存 Shape 产物：

```text
<task-workspace>/shape/
├── shape.md        # 只有方向比较、代价或关键未知需要交接时创建
└── evidence/       # 只有结论依赖探针、运行结果或外部材料时创建
```

`shape.md` 只记录真正改变方向的事实、仍有竞争力的路线、推荐及其代价、用户决定和剩余承重未知；可重放探针与原始材料进入 `evidence/`。

落盘后更新当前 Active Task 的 Current Work 和 Artifact Map，写明下一位应读取什么。没有 Active Task 时在对话中交付 Shape 结果，或只写用户指定的路径。

## 边界

Shape 只让方向变得值得承诺，不负责详细设计、实现、评审或测试，也不借调查扩大执行范围。
